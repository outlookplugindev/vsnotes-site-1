Artificial Intelligence

Author: Ashish Sinha · AI Developer
◆ Artificial Intelligence (AI) [1956]
AI is the grand goal of making machines behave intelligently like humans. It combines mathematics, data, and algorithms to simulate human thinking. AI does NOT mean robots only — most AI today runs in software systems. Real-world AI examples: ▸ Google Search ranking system ▸ ChatGPT answering questions ▸ Face recognition in phones ▸ Fraud detection in banking Major AI Domains (Beginner Friendly): ◇ NLP – Natural Language Processing (text, chat, search) NLP enables machines to work with human language (text and speech). Modern LLMs like ChatGPT are advanced NLP systems trained on large-scale data to understand context and generate responses.
📝 Text Input → ✂ Tokenization → 🧠 Embeddings → 🤖 Transformer / LLM → 💬 Output Response
Simple idea: Text → Tokenization → Understanding → Response Generation (LLM) Core tasks: • Text understanding (sentiment analysis, intent detection) • Language generation (chatbots, AI writing tools) • Translation (English ↔ Hindi, etc.) • Summarization (long text → short summary) Real examples: ▸ ChatGPT → conversational AI ▸ Google Translate → language conversion ▸ Gmail Smart Reply → auto responses How NLP works (simple flow): Text → Tokenization → Embeddings → Model → Output Challenges: • sarcasm understanding • context tracking • multiple meanings of same word ◇ Vision – Image & video understanding Vision AI helps machines “see” like humans. Core tasks: • object detection (cars, people, animals) • face recognition • image classification • medical image analysis Real examples: ▸ iPhone Face ID ▸ Tesla self-driving cameras ▸ Google Photos auto tagging Why important: → 80% of real-world data is visual ◇ Speech – Voice recognition, TTS Speech AI converts voice ↔ text. Tasks: • Speech-to-text (STT) • Text-to-speech (TTS) • voice command understanding Examples: ▸ Alexa / Siri ▸ YouTube auto captions ▸ call center voice bots Challenges: • accents • background noise • multiple speakers ◇ Robotics – Physical intelligent systems Robotics combines AI + hardware + sensors. Tasks: • movement control • object detection in real world • decision making in physical space Examples: ▸ self-driving cars ▸ warehouse robots (Amazon) ▸ surgical robots Key components: sensors → perception → AI model → motor action ◇ Recommender Systems – Netflix, YouTube These systems predict what you will like next. Types: • content-based filtering • collaborative filtering • hybrid systems Real examples: ▸ Netflix movie suggestions ▸ YouTube next video autoplay ▸ Amazon “customers also bought” How it works: user behavior → pattern learning → ranking → recommendation
◆ Rule‑Based / Symbolic AI
Humans explicitly write IF–ELSE rules. These systems follow fixed logical conditions created by experts. How it works: IF condition is true → THEN execute predefined action Example: IF temperature > 100°F THEN diagnose = fever IF user_input = "hello" THEN reply = "hi" Where it was used: • early chatbots (ELIZA style systems) • expert systems in medical diagnosis • industrial automation rules Limitation in real world: • cannot learn from new data • requires manual rule creation for every case • breaks when input is slightly different Failed because: • Rules explode in real life (combinatorial explosion) • Cannot handle uncertainty or probability • No adaptation ability • Hard to maintain Real-world insight: → Works only in controlled environments, not in dynamic environments like language, vision, or human behavior
◆ Machine Learning (ML) [1990s]
Machine Learning (ML) is a branch of AI where systems learn patterns directly from data instead of being explicitly programmed with rules. Core idea: → Instead of writing logic, we show examples and let the model learn Real-world meaning: • Email spam detection learns from past emails • Netflix learns from your watching history • Banks learn fraud patterns from transactions Learning loop: Data → Prediction → Loss → Backpropagation → Better Model Explanation of loop: • Data → input examples (images, text, numbers) • Prediction → model guesses output • Loss → difference between correct vs predicted answer • Backpropagation → adjusts internal weights to reduce error • Better Model → improved accuracy after training Why ML is powerful: • finds hidden patterns humans cannot easily see • improves automatically with more data • works for prediction-based problems Limitations of ML: • depends heavily on quality of data • struggles with raw unstructured data (images/audio/text) • needs manual feature engineering in many cases ◇ Learning Types ├─ Supervised Learning │ • Uses labeled data (input + correct output) │ • Model learns mapping between input and output │ Real examples: spam detection, image classification, price prediction. │ Key idea: → "Learn from correct answers" │ └─ Unsupervised Learning • Uses unlabeled data (no correct answers given) • Model finds hidden patterns by itself Real examples: customer segmentation, news grouping, anomaly detection. Key idea: → "Discover structure without guidance"
◆ Deep Learning (DL) [2012]
ML failed on raw images, speech and text → DL started. Reason for failure of ML: • Images have millions of pixel features → too complex for manual feature engineering • Speech is continuous time-series data → ML cannot capture sequence well • Text requires context understanding → ML cannot model long dependencies Deep Learning (DL) solved this by removing manual feature engineering and letting the model learn features automatically from raw data. Uses Neural Networks inspired by human brain. Idea behind Neural Networks: → mimic how neurons in the brain connect and pass signals → each neuron learns small patterns and combines them into complex understanding Why DL became powerful: • works directly on raw data • improves with more data and compute (GPU/TPU) • automatically builds feature hierarchy ◇ Neural Network Layers ├─ Input Layer → receives raw data │ • example: image → pixel values, text → token IDs ├─ Hidden Layers → learn patterns │ Layer 1 → simple patterns (edges) │ Layer 2 → shapes / phrases │ Layer 3 → complex objects / meaning └─ Output Layer → prediction Final decision. Probability-based: Cat = 0.82, Dog = 0.18 ◆ CNN (Vision Models) CNN is designed specifically for understanding images and visual data. How CNN “sees”: human brain (edges → shapes) ≈ CNN (filters → features). Key components: Convolution, ReLU, Pooling. ◆ Transformer [2017] Designed to process sequential data using Attention. Solves RNN/LSTM limitations (forgetting long context). Core idea: Looks at ALL words at once and decides which matter most.
◆ Large Language Models (LLMs) [2018+]
LLMs are trained on massive text to understand and generate human language. Core idea: Predict the next word (token) based on previous context.
Example Input: "I love you"
◇ Tokenization Breaking text into small units called tokens. Word Tokens: ["I", "love", "you"] Sub-word (BPE): ["I", "lo", "ve", "you"] Why? Handles unknown words, reduces vocab size. ◇ Vocabulary Fixed dictionary where each token has a unique numerical ID. "I" → 101, "love" → 458, "you" → 932. ◇ Embedding Layer Converts IDs into dense numerical vectors to give semantic meaning. ◇ Transformer Layers Self-Attention: every word looks at every other word. Feed-Forward: refines relationships. ◇ Vocabulary Projection Converts hidden vector → softmax → probability distribution. "you" = 0.62, "me" = 0.21. ◇ Generated Text Model generates text step-by-step. Key limitation: No real understanding, purely statistical patterns. ◆ RAG (Retrieval-Augmented Generation) Enhances LLMs by connecting them with external knowledge. ├─ Data Collection: Docs / PDFs / APIs ├─ Data Cleaning: Removes noise ├─ Chunking: Splits into meaningful pieces ├─ Vector Embedding: Numerical vectors for search ├─ Vector Database: AI memory (Azure AI Search / FAISS) ├─ Similarity Search: Finds "closest meaning" (Cosine Similarity) ├─ Prompt + Context: Adds retrieved chunks to user query └─ Grounded Answer: AI is grounded in real data, reduces hallucination.
